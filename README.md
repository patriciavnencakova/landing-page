# 🚀 The Odin Project: Landing Page

This repository contains my solution to the **Landing Page** project from the **Foundations** path of [The Odin Project](https://www.theodinproject.com/). The main goal of this assignment was to translate a comprehensive design mockup into a functional web page using only raw HTML and CSS.

---

## 💡 Skills & Concepts Practiced

This project was an excellent opportunity to transition from learning isolated concepts to building a cohesive, multi-section webpage from scratch. Here is a breakdown of what I practiced and masterered:

### 1. Advanced Flexbox Layouts 🧱
*   **Complex Alignment:** Positioned and aligned elements within the navigation bar, the hero section, and the multi-card layout using `flex-direction`, `justify-content`, and `align-items`.
*   **Distribution:** Managed spacing and content flow efficiently without relying on rigid, hardcoded pixel values.

### 2. Semantic HTML5 Structure 📄
*   Moved away from generic `<div>` soup to write meaningful, accessible code using semantic elements like `<header>`, `<nav>`, `<main>`, `<section>`, and `<footer>`.
*   Maintained a proper heading hierarchy (`<h1>` through `<h3>`) to align with web accessibility and SEO best practices.

### 3. CSS Fundamentals & Styling Precision ✨
*   **The Box Model:** Fine-tuned `padding` and `margin` values to ensure the spacing and proportions accurately match the provided design specs.
*   **Typography:** Handled font weight, text colors, and responsive text sizing to create a clean visual hierarchy.
*   **UI Components:** Designed interactive elements, such as styled action buttons with custom `border-radius`, background colors, and hover effects for better user experience.


## 📁 Repository Structure

*   `index.html` – The core semantic markup of the landing page.
*   `styles.css` – All layout and styling rules, organized logically into sections (Header, Hero, Info, Quote, CTA, Footer).
*   `images/` – Directory containing the asset images used throughout the page.
